EXPERIMENT 1

AIM

Explore the functionality and uses of the web browser in Web Technology which is helpful in designing and debugging.

REQUIREMENTS

A Web browser installed in the computer.

THEORY

Web Browser:An application software for accessing the World Wide Web or local website. In simple words it retrieves and shows the content from the web server to the user`s device.
Examples: Chromium, Firefox, Brave, Google Chrome, Safari etc...

Fig 1.1: Different Web browsers logos

Debugging: Debugging is a process to find and remove bugs that causes errors in proper functionality of software, apps or websites.

Debugger: A tool which helps in debugging code by running the program under controlled conditions. In general all debuggers consist of features like stopping execution of a program at a particular line also called breakpoint, step in and step out feature.
In the context of Web Technology, a debugger is useful in making proper designs and debugging the website flow like debugging interactions on the web page.

Web browser Debugger (Browser Dev Tools): The browser debugger or developer tools comes inbuilt in the browser. These tools make the development handy for web developers because it helps in tweaking styles, manipulating document object model, debugging javascript code, and checking the responsiveness of the website on different screen sizes and devices.

Fig 1.2: Browser Dev Tools


Steps to open debugger in browser:
  • Right click on the browser screen and click Inspect option.
  • Press F12.
  • Press ctrl+shift+I (windows or linux) or cmd+shift+I (mac).
  • Press ctrl+shift+J (windows or linux) or cmd+shift+J (mac).
  • Press ctrl+shift+K (windows or linux) or cmd+shift+K (mac).

    • DEBUGGER FEATURES

    1. Network Panel

      This dev tool panel consists of all the network requests that browsers make to retrieve web pages content from the web server. It consists of the lists of requests which are necessary to load the webpage which includes different types of files i.e. html, js, css, json, png, jpeg, plain etc...

      Fig 1.3: Network Panel with all network requests

      The network panel consists of a table which contains 9 columns:

      1. Status: It shows the status of the requests i.e. the HTTP status codes.
          - Informational responses (100-199)
          - Successful responses (200-299)
          - Redirection messages (300-399)
          - Client error messages (400-499)
          - Server error messages (500-599)
      2. Method: It shows the type of request i.e. GET, POST, PUT, PATCH, DELETE.
      3. Domain: It is the domain where the request is made.
      4. File: It consists of the complete URL or blob data of the file that was requested.
      5. Initiator: The source or tag which initiated the request.
      6. Type:The type of the file which is being requested i.e. html, css, js, json, png, etc...
      7. Transferred: The data which is used to get that file from web server i.e. 36.22KB.
      8. Size: After unzipping the actual size of the transferred data.
      9. Timeline: The time in milliseconds to get that file from the web server.

      There are other options also like disable cache (if true, it does not caches the retrieved file in the web browser) and throttling (triggers the internet speed to load content from the web server).

    2. Sources Panel

      This tab or panel consists of all the files that are required by the website for proper functioning.

      Fig 1.4: Sources Panel of the Dev tools in Browser

      This panel is mostly used for debugging the interactions in the websites i.e. javascript. It can also evaluate step by step execution of javascript code by use of breakpoints (XHR, Event Listener, DOM Mutation breakpoints) watch expressions and threads.

      Fig 1.5: Paused on breakpoint in javascript code

    3. Inspector or Elements Panel

      This panel consists of the html file of the web page and on inspecting the tag of the html webpage it also shows the styles, event listeners and dom objects attached to that tag.

      Fig 1.6: Inspect Panel of dev tools in browser

      It consists of searching HTML and picking element pointer-like features which helps in finding the actual tag inside the html content.
      It can change the style of the web page easily by adding inline style to the selected tag. This panel is most important when debugging design issues because it can easily tweak the style and html accordingly by clicking on the tag and adding the styles.
      It also shows the flexbox, grid or box model of the particular tag which helps in understanding the orientation of the tag in the window.
      To change the style just select the tag where the style needs to be applied and then in the style tab add the css properties which needs to be applied to get the desired design. It can also tweak the style of a css style class which will change all the style of the tag which consists of that style.

      Fig 1.7: Tweaking style of google logo by adding border

    4. Performance Panel

      This panel is important to analyze runtime performance of the website. From here we can change the CPU processor for different devices. It is important because it helps in optimizing the runtime performance of the website on different devices like old smartphones which have low cpu speed. This panel consists of an option to toggle the cpu speed, network speed and analyze the performance of a particular task also.

      Fig 1.8: Runtime performance of google.com
